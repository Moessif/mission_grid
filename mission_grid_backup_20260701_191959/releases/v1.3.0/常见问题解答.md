# MissionGrid 常见问题解答

## 概述

本文档整理 MissionGrid 地面站和任务包使用过程中的常见问题及解决方案。

---

## 1. 地面站相关

### Q1: 启动时报错 "No module named 'PySide6'"

**原因**：Python 依赖未安装

**解决方案**：
```bash
pip install PySide6 pymavlink python-tsp numpy
```

### Q2: 启动时报错 "No module named 'pymavlink'"

**原因**：pymavlink 未安装

**解决方案**：
```bash
pip install pymavlink
```

### Q3: 界面显示异常（按钮错位、文字重叠）

**原因**：DPI 缩放问题

**解决方案**：
1. 右键 `python.exe` → 属性 → 兼容性
2. 勾选"替代高 DPI 缩放行为"
3. 选择"系统"或"系统（增强）"

### Q4: 导出任务包时报错 "No path found"

**原因**：没有有效的路径（可能所有格子都是禁飞区）

**解决方案**：
1. 检查是否有禁飞区阻挡路径
2. 确保至少有一个非禁飞区格子
3. 尝试手动添加航点

### Q5: 遥测无法连接

**原因**：MAVLink UDP 连接失败

**解决方案**：
1. 确保无人机已开机并连接 MAVROS
2. 检查防火墙是否阻止 UDP 端口
3. 确认 IP 地址和端口配置正确

## 2. 任务包相关

### Q6: 任务包无法运行（Permission denied）

**原因**：Shell 脚本没有执行权限

**解决方案**：
```bash
chmod +x run_mission.sh
./run_mission.sh
```

### Q7: 运行时报错 "No module named 'uav_ctrl_tools'"

**原因**：Python 脚本路径问题

**解决方案**：
1. 确保在任务包目录下运行
2. 检查 `uav_ctrl_tools.py` 是否在 `/home/orangepi/ctrl_ws/src/competition_pkg/scripts/`

### Q8: 无人机不起飞

**检查项**：
1. ✅ 遥控器是否已解锁
2. ✅ MAVROS 是否连接成功（等待 30 秒）
3. ✅ 电池电量是否充足
4. ✅ SLAM 定位是否稳定

**日志查看**：
```bash
# 查看 MAVROS 连接状态
rostopic echo /mavros/state

# 查看 SLAM 定位
rostopic echo /Odometry
```

### Q9: 无人机原地打转

**原因**：航向角设置错误（v1.1.0 已修复）

**解决方案**：
- 更新到最新版本（v1.2.0+）
- 确保使用 `math.radians()` 转换角度

### Q10: 任务完成后在哪里降落

**逻辑**：
- **有降落动作**：在指定格子降落
- **无降落动作**：返回起飞点（默认 A9B1）降落

**查看起飞点**：
- 在 MissionGrid GUI 中查看"起飞"动作所在的格子
- 或查看 `mission_config.json` 中的 `takeoff_col` 和 `takeoff_row`

### Q11: 任务执行中途卡住

**可能原因**：
1. 网络中断
2. SLAM 定位丢失
3. 遥控器信号干扰

**解决方案**：
1. 遥控器切换到手动模式接管
2. 检查日志输出
3. 重启任务

## 3. 部署相关

### Q12: SCP 传输失败

**原因**：网络连接问题

**解决方案**：
1. 确保电脑和 OrangePi 在同一网络
2. 检查 SSH 服务是否开启：
   ```bash
   sudo systemctl status ssh
   ```
3. 尝试使用 USB 拷贝

### Q13: OrangePi 无法 SSH 连接

**检查项**：
1. OrangePi 是否开机
2. IP 地址是否正确
3. SSH 服务是否开启

**开启 SSH**：
```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

### Q14: 任务包文件损坏

**原因**：传输过程中断

**解决方案**：
1. 重新生成任务包
2. 重新传输
3. 使用校验和验证：
   ```bash
   # Windows
   Get-FileHash mission_grid_YYYYMMDD_HHMMSS -Algorithm MD5
   
   # OrangePi
   md5sum mission_grid_YYYYMMDD_HHMMSS/*
   ```

## 4. 环境配置

### Q15: PyTorch 报错 "libgomp.so.1: cannot open shared object file"

**原因**：缺少 OpenMP 库

**解决方案**：
```bash
export LD_PRELOAD=/home/orangepi/.local/lib/python3.8/site-packages/torch/lib/libgomp-d22c30c5.so.1
```

**永久解决**：添加到 `~/.bashrc`：
```bash
echo 'export LD_PRELOAD=/home/orangepi/.local/lib/python3.8/site-packages/torch/lib/libgomp-d22c30c5.so.1' >> ~/.bashrc
source ~/.bashrc
```

### Q16: ROS 环境未加载

**原因**：未 source ROS 工作空间

**解决方案**：
```bash
source /opt/ros/noetic/setup.bash --extend
source /home/orangepi/tools_ws/devel/setup.bash --extend
source /home/orangepi/livox_ws/devel/setup.bash --extend
source /home/orangepi/ctrl_ws/devel/setup.bash --extend
```

### Q17: SLAM 启动失败

**原因**：激光雷达驱动未启动

**解决方案**：
```bash
# 先启动激光雷达
roslaunch livox_ros_driver2 msg_MID360s.launch &
sleep 10

# 再启动 SLAM
rosrun competition_pkg node_manage.py &
```

## 5. 调试技巧

### 5.1 查看 ROS Topic

```bash
# 列出所有 Topic
rostopic list

# 查看 Topic 数据
rostopic echo /mavros/state
rostopic echo /Odometry
rostopic echo /camera/color/image_raw
```

### 5.2 查看节点状态

```bash
# 列出所有节点
rosnode list

# 查看节点信息
rosnode info /mavros
```

### 5.3 查看日志

```bash
# 查看 ROS 日志
rosrun rqt_logger_level rqt_logger_level

# 查看系统日志
journalctl -u ros -f
```

### 5.4 手动测试控制

```bash
# 进入 Python 交互环境
python3

# 测试控制接口
import uav_ctrl_tools as ctrl_tools
uav = ctrl_tools.CtrlTools()
uav.uav_takeoff()
uav.set_point(0, 0, 1.2)
uav.uav_land()
```
