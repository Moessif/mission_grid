# 无人机任务编排器

这是一个运行在 Windows 上的图形化程序，用来把无人机的大任务拆成基础小任务，再组合导出成新的机载任务包。

它当前以你桌面上的这套环境为参考：

- `C:\Users\王震\Desktop\无人机计算机`
- `C:\Users\王震\Desktop\无人机资料`
- `C:\Users\王震\Desktop\无人机已成功脚本`

它的原则不变：

1. 不覆盖你原来的机载脚本。
2. 优先复用你已经跑通过的控制和识别代码。
3. 只导出当前这套资料里能真正落地的任务。

也就是说，任务不再写死成 H。

## 5. 图形界面怎么用

### 最简单的方式

1. 在中间上方选一个“快速模板”。
2. 点“套用模板”。
3. 选中中间某个任务。
4. 在右边改参数。
5. 点“导出任务包”。

### 自己手动拼

1. 左边双击任务，加入到中间流程。
2. 用“上移 / 下移”调顺序。
3. 用右边参数区修改任务内容。
4. 在右侧“导出校验”看是否可导出。

### 常用快捷按钮

中间区域还加了几个一键按钮：

- 加入起飞
- 加入返航
- 加入降落
- 加入目标识别

## 6. 新的可编辑目标怎么用

如果你想从“降落到 H”改成“降落到 A”或者别的目标，可以这样做：

1. 工作流里加入：
   - `识别视觉目标标识`
   - `计算目标标识坐标`
   - `降落到目标标识`
2. 在 `识别视觉目标标识` 参数里修改：
   - `marker_type`
   - `marker_label`
   - `marker_qr_text`

例如：

- `marker_type = letter_h`，`marker_label = H`
- `marker_type = letter_a`，`marker_label = A`
- `marker_type = circle`
- `marker_type = qr_text`，再填 `marker_qr_text`

## 8. 导出后怎么上机

1. 把导出的整个文件夹复制到机载电脑。
2. 在机载电脑终端进入该目录。
3. 给入口脚本执行权限：

```bash
chmod +x run_utility_mission.sh
```

或者：

```bash
chmod +x run_inventory_mission.sh
```

4. 运行：

```bash
./run_utility_mission.sh
```
