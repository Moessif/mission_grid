## 4. 在 OrangePi 上测试（实际飞行）

如果测试脚本正常，可以运行实际任务脚本：

### 4.1 添加执行权限

```bash
chmod +x run_mission.sh
```

### 4.2 运行任务

```bash
./run_mission.sh
```

这个脚本会：
1. 启动 ROS 环境
2. 启动 MAVROS、激光雷达、摄像头
3. 启动 SLAM
4. 运行任务脚本

### 4.3 执行过程

1. **等待遥控器解锁**（你需要操作遥控器）
2. **起飞**
3. **按路径执行任务**
4. **返回起飞点降落**

## 5. 常见问题

### Q1: 测试脚本报错 "No module named 'uav_ctrl_tools'"

**原因**：测试脚本尝试导入 uav_ctrl_tools 模块

**解决方案**：测试脚本应该不会导入这个模块，如果出现这个错误，请检查脚本内容。

### Q2: 实际任务脚本报错 "RC unlock failed"

**原因**：遥控器未解锁

**解决方案**：
1. 确保遥控器已连接
2. 按照遥控器说明解锁

### Q3: 任务脚本报错 "Takeoff failed"

**原因**：起飞失败

**解决方案**：
1. 检查电池电量
2. 检查遥控器信号
3. 检查 MAVROS 连接

### Q4: 任务完成后在哪里降落

**逻辑**：
- 如果设置了降落动作 → 在指定格子降落
- 如果未设置降落动作 → 返回起飞点（默认 A9B1）降落

## 6. 调试技巧

### 6.1 查看 ROS 日志

```bash
# 查看 MAVROS 状态
rostopic echo /mavros/state

# 查看位置
rostopic echo /mavros/local_position/odom

# 查看 SLAM
rostopic echo /Odometry
```

### 6.2 手动测试控制

```bash
# 进入 Python 交互环境
python3

# 测试控制接口
import uav_ctrl_tools as ctrl_tools
uav = ctrl_tools.CtrlTools()
# 注意：以下命令会实际飞行
# uav.uav_takeoff()
# uav.set_point(0, 0, 1.2)
# uav.uav_land()
```

### 6.3 检查任务配置

```bash
# 查看任务配置
cat mission_config.json

# 查看任务脚本
cat generated_mission.py
```

## 7. 安全注意事项

⚠️ **重要安全提示**：

1. **首次测试请在空旷区域进行**
2. **确保遥控器随时可以接管**
3. **电池电量充足（建议 > 80%）**
4. **测试前检查所有连接**
5. **如有异常立即切换手动模式**
