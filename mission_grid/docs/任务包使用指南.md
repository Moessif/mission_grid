# MissionGrid 任务包使用指南

## 概述

MissionGrid 生成的任务包是可直接部署到 OrangePi 机载电脑的自动化飞行任务。本指南详细说明如何将任务包从地面站传输到无人机并执行。

---

## 1. 生成任务包

### 1.1 启动地面站

```bash
cd mission_grid
python app.py
```

### 1.2 编排任务

1. **设置起飞点**
   - 点击某个格子（默认 A9B1）
   - 选择"起飞"动作

2. **添加任务动作**
   - 左键点击格子 → 选择动作类型
   - 可选动作：拍照、二维码扫描、YOLO 检测、蜂鸣器、舵机等

3. **设置降落点**（可选）
   - 如果不设置：任务完成后自动返回起飞点降落
   - 如果设置：在指定格子直接降落

4. **标记禁飞区**
   - 右键点击格子 → 标记为禁飞区

5. **规划路径**
   - 点击"自动规划"按钮
   - 或手动添加航点

6. **导出任务包**
   - 点击"导出"按钮
   - 任务包保存在 `mission_grid/output/` 目录

---

## 2. 任务包结构

```
mission_grid_YYYYMMDD_HHMMSS/
├── generated_mission.py    # Python 任务脚本
├── run_mission.sh          # Shell 启动脚本
└── mission_config.json     # 航点和动作配置
```

### 2.1 generated_mission.py

- 使用 `uav_ctrl_tools.CtrlTools` 接口控制无人机
- 包含坐标转换（网格坐标 → MAVROS 坐标）
- 包含触发条件判断逻辑
- 包含降落逻辑（自动返回起飞点或指定降落点）

### 2.2 run_mission.sh

- 自动 source ROS 和所有工作空间
- 按顺序启动 ROS 节点（MAVROS → 激光雷达 → 摄像头）
- 启动 SLAM 并等待定位稳定
- 运行 Python 任务脚本

### 2.3 mission_config.json

- 记录所有航点坐标和动作配置
- 用于调试和任务复现

---

## 3. 部署到 OrangePi

### 3.1 方法一：SCP 传输（推荐）

```bash
# 在 Windows 终端执行
scp -r mission_grid/output/mission_grid_YYYYMMDD_HHMMSS orangepi@192.168.144.xxx:/home/orangepi/
```

### 3.2 方法二：USB 拷贝

1. 将 `mission_grid_YYYYMMDD_HHMMSS` 文件夹拷贝到 U 盘
2. 插入 OrangePi
3. 复制到 `/home/orangepi/` 目录

---

## 4. 在 OrangePi 上运行

### 4.1 SSH 连接

```bash
ssh orangepi@192.168.144.xxx
```

### 4.2 执行任务

```bash
# 进入任务目录
cd /home/orangepi/mission_grid_YYYYMMDD_HHMMSS

# 添加执行权限
chmod +x run_mission.sh

# 运行任务
./run_mission.sh
```

### 4.3 执行过程

`run_mission.sh` 会自动执行以下步骤：

1. **环境准备**（约 60 秒）
   - Source ROS 和所有工作空间
   - 启动 MAVROS（等待 30 秒连接飞控）
   - 启动 Livox 激光雷达
   - 启动摄像头
   - 启动 SLAM（等待定位稳定）

2. **任务执行**
   - 等待遥控器解锁（你需要操作遥控器）
   - 起飞
   - 按路径依次执行每个航点的动作
   - 完成后自动降落

## 5. 注意事项

### 5.1 飞行前检查

- ✅ 电池电量充足（建议 > 80%）
- ✅ 遥控器已连接并解锁
- ✅ GPS 信号良好（室外）或 SLAM 定位稳定（室内）
- ✅ 飞行区域无障碍物

### 5.2 紧急情况

- **立即接管**：遥控器切换到手动模式
- **紧急降落**：遥控器触发降落
- **任务中断**：Ctrl+C 终止脚本

### 5.3 日志查看

运行时终端会显示实时日志，包括：
- 当前航点位置
- 动作执行状态
- 错误信息

## 6. 常见问题

### Q1: 任务包无法运行

**检查项**：
1. 是否已添加执行权限：`chmod +x run_mission.sh`
2. Python 脚本路径是否正确
3. ROS 环境是否正确 source

### Q2: 无人机不起飞

**检查项**：
1. 遥控器是否已解锁
2. MAVROS 是否连接成功（等待 30 秒）
3. 电池电量是否充足

### Q3: 无人机原地打转

**原因**：航向角设置错误（v1.1.0 已修复）

### Q4: 任务完成后在哪里降落

**逻辑**：
- 如果设置了降落动作 → 在指定格子降落
- 如果未设置降落动作 → 返回起飞点（默认 A9B1）降落

## 7. 版本要求

- **OrangePi 系统**：Ubuntu 20.04
- **ROS 版本**：Noetic
- **Python 版本**：3.8
- **依赖包**：uav_ctrl_tools, uav_img_tools, rospy, cv2
